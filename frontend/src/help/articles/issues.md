# Issues

The Issues screen combines issue submission and approval workflows in one place. Users can submit bug reports, feature requests, and questions; reviewers can filter to pending approvals and approve or deny directly from an issue dialog.

## How It Works

1. **Open Issues.** The default view shows your issue requests.
2. **Use top filters.** Filter by search text, type, and status.
3. **Needs Approval only.** Reviewers can toggle this to only show requests awaiting approval.
4. **Create issue.** Use **New Issue** to open the form and submit a new request.
5. **Approve/Deny in dialog.** For pending requests, click the review icon on a card to open the approval dialog.

## Key Controls

| Element | Description |
|---|---|
| **New Issue** | Opens the issue submission form. |
| **Search** | Filters by title, area, or submitter. |
| **Type filter** | Shows bug reports, feature requests, questions, or all types. |
| **Status filter** | Shows pending, approved, rejected, or all statuses. |
| **Needs Approval Only** | Reviewer-only toggle to show pending requests needing action. |
| **Review icon** | Opens approve/deny dialog for a pending request. |
| **Approve** | Approves request and creates corresponding GitHub issue. |
| **Deny** | Rejects request with optional notes. |

## Tips

- Provide complete steps and expected vs actual behavior for bug reports.
- Use status + search filters together to quickly find older requests.
- Reviewers should add denial notes to help submitters correct and resubmit.
